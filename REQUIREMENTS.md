# REQUIREMENTS

## 1. アプリケーション概要
- 名称: MyGarden Diary
- 目的: ユーザが日付ごとに植物観察記録を残し、画像とメモを管理できる
- 対応環境: Webブラウザのみ（モバイル優先）

## 2. 主要機能
- カレンダー画面
- 新規記録作成
- 記録詳細表示
- 記録編集
- 同日複数記録一覧
- 画像圧縮設定
- 植物種マスタ管理（新規/編集画面のボトムシートで追加・編集）

## 3. 記録データ仕様
- 日付: 必須、初期値はJST基準の当日
- 植物の種類: 任意（plantTypeId）
- メモ: 任意、200文字以下
- 画像: 任意、複数枚添付可能

## 4. 画像アップロード仕様
- クライアント側で圧縮してからアップロードする
- 初期圧縮設定: 640x480 / 品質80%
- 圧縮設定は設定画面で変更可能
- サーバは圧縮済み画像を data/uploads に保存する
- 画像配信は /api/uploads/[fileName] 経由で行う

## 5. カレンダー仕様
- ルートアクセス時は /calendar に遷移
- カレンダーセルは日付エリアとメディアエリアを分離し、表示有無で高さが変化しない
- セルに記録がある場合:
- 画像あり: 最大3枚までサムネイルを重ねて表示し、背面ほど透過率を下げる
- 画像なし: Heroicons chat-bubble-left-ellipsis を表示
- 同日に複数件ある場合は日付別一覧へ遷移
- 1件のみの場合は詳細画面へ遷移
- 記録がない日付セルをタップすると、日付が選択済みの新規記録画面へ遷移
- 未来月は表示・遷移できないように制御する

## 6. 植物種管理仕様
- 選択方式で記録フォームから選択できる
- 新規/編集画面のボトムシートで追加・編集可能
- 削除は不可、アーカイブのみ可能
- アーカイブ済みは通常選択から非表示

## 7. UI/UX仕様
- モバイルファースト、レスポンシブ対応
- Heroiconsを利用
- ヘッダ文言は "MyGarden Diary"
- 緑基調のデザインと可読性を重視した文字色設計
- 設定画面はハンバーガーメニューのプルダウン経由で遷移
- ヘッダタイトルは左右アイコンの有無に影響されず常に中央に配置する
- カレンダー当日セルは外枠を使わず背景色のみで強調表示する
- 記録日数/記録数はカレンダー下部に表示する
- 新規記録導線はカレンダー画面ヘッダー左側の + ボタンに配置する
- iPhoneでの操作性を優先し、ダブルタップ拡大を抑制する
- iPhoneの小さい画面でも操作できるよう、ヘッダ上下余白を最小限にする
- iPhoneでも月切り替えボタンが操作可能なタップ領域を確保する
- 非カレンダー画面の戻るはブラウザ履歴依存にせず、画面導線として定義した遷移先へ戻す
- 記録詳細画面では画像タップで全画面表示できる
- 記録詳細画面には編集と削除の両操作を配置する
- 新規記録画面の植物種は「未分類」が存在する場合、初期選択を「未分類」にする

## 8. 保守性要件
- 共通UIコンポーネント化を行う
- 1ファイル150行未満を維持する
- 仕様変更時は本ファイルを更新する

## 9. 非機能要件
- Raspberry Pi運用を想定し、画像容量を抑制
- サーバ側は軽量なJSONストレージ構成
- 観察記録データとアップロード画像は git 管理対象外
